# Release audit manifest ‚Äî 2026-07-06

Findings from the six-track pre-release audit (movement fidelity, server
physics, Lua-vs-C semantics, protocol/networking, QuakeC VM, performance),
each produced by reading the Luau and the C reference side by side. Same
status discipline as the coverage manifests (see README.md): a row is done
only at **VERIFIED** (named test check, compiled-C truth, or committed
screenshot under evidence/), **SUBSTITUTED** requires a stated
justification, and evidence lands in the same commit as the fix.

Severity: **CRIT** changes how the game plays or can take the server down;
**MOD** wrong behavior in reachable scenarios; **MIN** edge cases /
latent; **PERF** performance before release.

## Audit non-findings kept for the record

Verified exact by the audit (no row needed): all pmove constants/order/
hull tracing incl. documented id quirks; SV_RunCmd msec chopping and the
no-maxvelocity-for-players QW rule; wire quantization C-truncation
(mathlib.trunc); NQ entity physics bug-for-bug (SOLID_BSP onground quirk,
corpse box collapse, the 215 chase diagonal); VM opcode semantics (float
truth, logical AND/OR, signed jumps, edict-reuse full zero); netchan-lite
substitution internally consistent on the ordered reliable remote; msg/
bufio already buffer-based end-to-end; particlesim fixed pool.
