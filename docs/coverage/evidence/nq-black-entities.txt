Black alias entities — non-pow2 TextureContent (2026-07-04)
============================================================

USER PLAYTEST REPORT: monsters and corpses rendering as solid black
silhouettes (alive and dead), world lit normally.

DIAGNOSTIC SEQUENCE (all on the live corpse at e1m1 q(31,1391), part

against the offline parse, whose BMP dump reproduces the authentic
soldier skin):

                                    red (texture pipeline works)
  nq-black-entities-pow2checker.jpg 256x256 checker -> renders with
                                    CORRECT UV wrapping

ROOT CAUSE: MeshPart TextureContent silently renders BLACK for
non-power-of-two EditableImages. Every Quake alias skin is non-pow2
(soldier 300x194, dog 312x169, flame 296x140, missile 288x195...);
world textures are pow2, which is why only entities went black. A
re-upload of identical pixels does not help; pixels read back
correctly (ReadPixelsBuffer is CPU-side). Note: destroyed
EditableImages report Size (1,1) — a useful liveness probe.

FIX: textures.createImagePow2 pads skins to the next power of two and
entrender scales UVs by (w/padW, h/padH); translated player skins take
the same path, with creation failures cached (the old code retried a
failed translation every frame, hammering the image budget).

VERIFY: boot NQ, approach any monster or corpse — skins render. The
GL/shotgun view models (same padded pipeline through the ViewportFrame
gun path) render textured post-fix in this session's captures.
